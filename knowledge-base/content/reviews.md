# Kundenstimmen — Lage und Regeln

**Stand: 2026-07-08**

## Crawl-Ergebnis

| Portal | Ergebnis |
|---|---|
| Google Maps | Bewertungen existieren laut Eintrag, sind aber nicht zuverlässig crawlbar; keine wörtlichen, attribuierbaren Zitate gewinnbar |
| golocal.de | 0 Bewertungen |
| Yelp.de | 0 Bewertungen („Nicht übernommen" — Eintrag unbeansprucht) |
| 11880.com | Seite blockt Direktabruf; Such-Snippets deuteten vereinzelte, gemischte Stimmen an — **nicht attribuierbar, nicht verwendbar** |
| friseurempfehlung.de | Server nicht erreichbar |

**Konsequenz:** Es gibt derzeit **keine einzige verwendbare, verifizierte Kundenstimme.**

## Regeln (bindend)

1. **Niemals Zitate erfinden.** Auch nicht „typische" oder „zusammengefasste" Stimmen. (Hausregel aus PROMPT.md.)
2. Die Testimonial-Sektion der Website ist gebaut, rendert aber `null`, solange `app/data/testimonials.ts` leer ist. Kein sichtbarer Platzhalter, keine Lücke im Layout.
3. Aufnahme einer Stimme erfordert: wörtliches Zitat + Name (oder freigegebenes Kürzel) + schriftliches Einverständnis + Quelle/Datum hier im KB.

## Wie echte Stimmen beschafft werden (nach Go-Live-Gespräch)

- Anetta fragt Stammkundinnen und -kunden direkt um Erlaubnis (der glaubwürdigste Weg bei einem Nachbarschaftssalon).
- Google-Bewertungen: Kundin/Kunde um Erlaubnis bitten, die eigene Google-Rezension zitieren zu dürfen (Screenshot + Einverständnis archivieren).
- Jede freigegebene Stimme wird hier als Eintrag ergänzt: Zitat · Name/Kürzel · Quelle · Datum · Einverständnis-Nachweis.
